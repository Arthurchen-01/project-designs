## 一、项目愿景

这不是一个简单的数据看板，而是一个**AI驱动的AP备考平台**：
- 背后由大模型（API接入）做智能调控和评估
- 每天动态更新，根据输入自动调整5分率预测
- 同时是**班级资源分享平台**，同学间可以交流资源
- 可推广给其他班级使用

## 三、核心功能：AI驱动的5分率系统

### 3.1 核心设计理念

- **AI判断，非公式计算**：5分率由大模型根据多维输入综合判断
- **严格标准**：AI以比官方更严格的标准评估，宁可低估不可高估
- **记忆曲线意识**：不复习就自然衰减，复习才有机会增长
- **输入模糊度影响评分**：描述越模糊，AI给的增幅越小

### 3.2 5分率评估逻辑

#### 输入来源（学生每天更新）

| 输入类型 | 内容 | 示例 |
|----------|------|------|
| 测试类 | MCQ/FRQ 模考成绩 + 用时 | "今天做了FRQ，100分题拿了82分，计时完成" |
| 复习类 | 描述复习了什么 | "复习了宏观经济的AD-AS模型" |
| 刷题类 | 刷题的数量和质量 | "做了30道MCQ，错了5道" |
| 其他 | 任何相关学习活动 | "看了一个AP FRQ的视频讲解" |

#### AI评估规则

```
每天的5分率变化 = f(历史表现, 近期趋势, 输入质量, 记忆曲线)
```

**具体规则：**

1. **有测试成绩**：AI综合以下因素评估
   - 历史成绩占比 ~60%（长期稳定表现）
   - 近期趋势占比 ~40%（上升/下降方向）
   - 成绩稳定性（波动越小越好）
   - 例如：60分及格线，你考了55→60→77，趋势上升，AI给60%-70%的5分率

2. **无测试，只有复习描述**：AI根据描述深度判断增幅
   - **深度描述**（"把AD-AS模型的每个细节都重新梳理了一遍"）→ 较大增幅
   - **模糊描述**（"复习了"三个字）→ 最小增幅，甚至可能仍是微降
   - **原因**：记忆曲线天然衰减，只有有效复习才能抵消或逆转

3. **什么都不填**：直接衰减（遗忘曲线）
   - 不复习 = 知识在遗忘，5分率应自然下降

4. **特殊情况**：
   - 稳定满分学生（如每次都80/60及格线）：即使当天只说了"复习了"，5分率几乎不变
   - 原因：已掌握透彻，边际收益递减

#### 示例场景

> 满分5分，及格线60分。某同学3次模考：55、60、77
> 
> AI分析：
> - 历史：均值64，略高于及格线，中等 → 占60%权重
> - 趋势：55→60→77，明显上升 → 占40%权重
> - 综合判断：**5分率 65-70%**
> - 如果他今天只说"复习了" → +0.5%（微增）
> - 如果他详细描述了知识点 → +1-3%
> - 如果他什么都没填 → -1-2%（遗忘）

### 3.3 AI评判标准说明

| 场景 | AI行为 |
|------|--------|
| 详细测试 + 高分 | 5分率显著提升 |
| 详细测试 + 中等分 | 5分率小幅提升 |
| 详细复习描述 | 小幅提升 |
| 模糊复习（"复习了"） | 极小幅提升或持平 |
| 什么都不填 | 自然衰减 |
| 长期稳定高分 + 模糊输入 | 几乎不变 |

## 四、页面详细设计

### 4.1 首页（Landing Page）
- 功能：选择/进入班级

### 4.3 考试日历（Calendar）
- 仅5月
- 每天方块较大，颜色清晰
- 每日显示：距今天数 / 科目+报名人数 / 平均5分概率

### 4.4 个人中心（Personal）
- 个人仪表盘（可从班级点入或直接进入）
- 5分率趋势图（每日波动曲线）
- MCQ/FRQ得分趋势
- **📝 更新信息**（核心入口）
  - 点击后弹出/跳转到当日信息填写表单
  - 见下方表单设计

### 4.5 每日更新表单设计（关键交互）

学生点击「更新信息」后填写：

```
┌─────────────────────────────────────┐
│         📝 今日学习更新              │
├─────────────────────────────────────┤
│                                     │
│  类型：                              │
│  ○ MCQ测试   ○ FRQ测试              │
│  ○ 知识点复习  ○ 刷题  ○ 其他        │
│                                     │
│  （如选测试类）                      │
│  分数：_____ / 满分_____             │
│  是否计时：○ 是  ○ 否               │
│                                     │
│  具体内容描述：                      │
│  ┌─────────────────────────────┐   │
│  │ 描述今天具体做了什么…        │   │
│  │ （越详细，AI评估越准确）     │   │
│  └─────────────────────────────┘   │
│                                     │
│        [ 提交更新 ]                  │
└─────────────────────────────────────┘
```

**设计要点**：
- 描述框下方提示"越详细，AI评估越准确"
- 支持快速模式（选类型+填分数）和详细模式（附文字描述）
- 提交后 AI 立即重新计算5分率并展示变化

### 4.6 资源分享（Resource Sharing）
