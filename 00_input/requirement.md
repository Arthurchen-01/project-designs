# 需求更新：苏格拉底家教系统完整方案

- 提出者：GuanYu
- 时间：2026-03-30

## 一、核心需求总结

### 1. Repository 管理
- 开一个 Git repository，存放所有学习文件（角色设定、进度、日记等）
- 仓库就是"记忆"的唯一来源

### 2. 一键启动（复制粘贴方案）
- 每次开始学习时，只需复制粘贴一个东西给网页版 Gemini/GPT
- 这个"东西"应该是一个链接或一段文本，让 AI 知道：
  - 它要扮演什么角色
  - 当前学习进度在哪里
  - 之前的记忆（长期+中期+短期）
- **早上**：需要长期记忆 + 近期进度
- **晚上（复习）**：只需要当天的详细记录 + 角色设定

### 3. 问题收集与 Anki 导出
- **所有问题必须收集**（不只是 diary/progress 里的总结）
- 学习结束后，一键保存当天所有问题和回答
- 生成符合 `word.docx` 排版规范的 DOCX 文件
- 用户的工具可以直接把 DOCX 转成 Anki

### 4. 课后自动更新
学习结束后，自动更新 repository 中的：
- progress.md（进度）
- diary.md（日记）
- 角色人设文件（关系变化）
- mid_term_memory.md（中期记忆压缩）
- wechat_unread.md（群聊）
- session_archive.md（归档旧记录）

## 二、Anki DOCX 格式规范（来自 word.docx）

### 牌组和标签
- `#` = 子牌组名称
- `TAG` = 标签（可多个）

### 题型格式
- `* 单选题` — 选项 A/B/C/D，答案用括号标注
- `* 多选题` — 答案可以括号内或下方 `答案: XX`
- `* 判断题` — A.对 B.错，`答案为:X`
- `* 简答题` — 题目一段，`答案：`后跟内容
- `* 名词解释` — 同简答格式
- `* 填空题` — 挖空用颜色 C00000（红色）
- `* 批注` — 文本 E97132，内容 00B0F0
- `* 折叠` — 标题 F64926，内容 0E2841
- `* 隐藏` — 标签 CF4DAA，内容 3A7C22

### 颜色代码表
| 颜色 | 用途 |
|------|------|
| C00000 | 填空挖空 |
| E97132 | 批注文本 |
| 00B0F0 | 批注内容 |
| F64926 | 折叠标题 |
| 0E2841 | 折叠内容 |
| CF4DAA | 隐藏标签 |
| 3A7C22 | 隐藏内容 |

## 三、技术方案

### Phase 1: Repository + Prompt 生成器
1. 将 `Sakura - gemini版/` 迁移为 Git repository
2. 创建一键 prompt 生成脚本（支持早上/晚上模式）
3. 使用 GitHub/Gitee raw 链接作为"一个链接"方案

### Phase 2: 问题收集 + Anki DOCX 生成
1. 对话记录解析器（从 AI 对话中提取 Q&A）
2. DOCX 生成器（按 word.docx 规范输出）
3. 一键导出流程

### Phase 3: 自动化流程（课后更新）
1. 脚本自动更新所有 md 文件
2. Git 自动 commit + push
