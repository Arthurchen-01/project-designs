**Type**: Technical cleanup

### 2. ❌ N+1 query in advice API (MEDIUM risk)
`src/app/api/ai/advice/route.ts` — loops over `student.subjects`, then for each subject:
- Calls `probabilitySnapshot.findMany()` 
- Calls `assessmentRecord.findMany()`

This is sequential N+1. With 8 AP subjects, that's ~16 DB queries per advice request.

**Type**: Technical / performance
**Impact**: Slow advice loading (acceptable for V1, needs fix before scale)

### 3. ⚠️ confidence.ts semantics gap (MEDIUM risk)
`getConfidenceLevel()` uses only **record count** (≥5 = high, ≥2 = medium). It ignores:
- Recency of records
- Consistency of data
- AI evaluation quality

A student with 5 records from 6 months ago gets "high confidence" — misleading.

**Type**: Product / user-facing
**Impact**: Students may over-trust stale data

### 4. ⚠️ Weak units hardcoded as "核心概念" (MEDIUM risk)
`ai-advisor.ts` line ~72: `if (latestRate < 60) weakestUnits.push('核心概念')`. This is a placeholder — actual unit-level analysis requires topic tagging in assessment records.

**Type**: Product / user-facing
**Impact**: AI advice is generic, not actionable for specific unit weaknesses

### 5. ❌ No error boundary in personal page charts (LOW risk)
`personal/[subjectId]/page.tsx` renders Recharts `LineChart` and `BarChart` but has no error boundary. If snapshot data is malformed, the page crashes.

**Type**: Technical / UX
**Impact**: White screen on data issues
