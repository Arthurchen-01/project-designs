## Bug Analysis: 角色文件持久化

**根因:** `update_role_files()` 中，当 `RELATION_SECTION` 不在文件中时：
```python
if RELATION_SECTION not in role_content:
    new_section = f"\n\n{RELATION_SECTION}\n\n- **{date_str}** — {sentiment}"
else:
    ...
```
`new_section` 被构建但从未拼接到 `role_content`，导致写入的内容是原始未修改的文件。

**影响:** 首次检测到老师互动时关系段落不会写入。后续运行（段落已存在）走 else 分支正常工作。

**严重性:** 低 — 不影响核心功能（progress/diary/archive/memory 更新），仅影响角色关系记录的首次持久化。
