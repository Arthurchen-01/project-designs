## 1. Beta 更新逻辑 ✅ 正确

| PRD 要求 | 实现 | 状态 |
|-----------|------|------|
| 先验 α₀=2, β₀=2 | ALPHA_0=2, BETA_0=2 | ✅ |
| 每次测试贡献 success = w·m·q | `successEvidence += w * m * q` | ✅ |
| 每次测试贡献 fail = w·m·(1-q) | `failEvidence += w * m * (1-q)` | ✅ |
| α = α₀ + Σsuccess | `alpha = ALPHA_0 + successEvidence` | ✅ |
| β = β₀ + Σfail | `beta = BETA_0 + failEvidence` | ✅ |

- 默认 slope=7，在 PRD 建议的 6-8 范围内 ✅

## 发现的问题

### ⚠️ MEDIUM — 遗忘衰减与区间不联动

**问题描述：**  
代码先算出原始 rate 和 CI，然后对 rate 施加衰减 `(rawRate * (1 - decay))`，但 CI 仅做平移（`lower0 + rateShift`），宽度不变。

**影响：**  
当衰减明显时（如 10-15%），点估计下降了，但区间宽度不变。这暗示系统"对衰减后的值仍然同样确定"，实际上衰减意味着证据减少，不确定性应增大（区间应变宽）。

**建议修复：**  
衰减应同时影响 effectiveSampleSize（或等效地缩小 alpha+beta），让区间自然变宽，而非仅平移。

### ⚠️ LOW — 遗忘衰减作用对象有歧义

PRD 描述"完全没更新：小幅下降"，代码将衰减应用为 `rawRate * (1-decay)`。这意味着：
- rate=0.80 时，15% 衰减 → 0.68（绝对下降 12%）
- rate=0.30 时，15% 衰减 → 0.255（绝对下降 4.5%）

这是"比例衰减"而非 PRD 中暗示的"小幅微调"。在高 5 分率区间效果偏大。建议改为加法衰减（如 rate -= decay * factor），或衰减作用于 evidence 而非最终 rate。

### ⚠️ LOW — 趋势估计使用原始分数而非 logistic 支持度

`estimateTrend` 使用 `score/maxScore` 的线性回归斜率，而非 `logisticSupport` 变换后的值。这在分数接近 cutoff 时可能导致趋势方向判断不一致（如 58→62 在 raw 上仅 +4%，但在 logistic 上可能从 0.36 跳到 0.57，趋势更显著）。

### ℹ️ INFO — 有效样本量计算

`effectiveSampleSize = alpha + beta - ALPHA_0 - BETA_0 = alpha + beta - 4`，即扣除先验后的总证据量。逻辑正确，但需确保 UI 层不会将此值误解为"测试次数"。
