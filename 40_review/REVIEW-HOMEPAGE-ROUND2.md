**审查人**: Agent 3 (Round 2)

## 1. Hero 口号冲击力评估

**当前口号**: "用拳头书写每一页历史"

> 注意：第一轮审查中提到的"每一场胜利，都是答案"已被替换为当前版本。

### 评价

当前版本比上一版好——"拳头"+"历史"有画面感，但仍有问题：

- **"每一页"拖慢节奏**。口号应该是一拳打出去的感觉，"每一页"像翻书，削弱了力量感。
- **缺少身份感**——谁在用拳头写？平台？选手？没有主语，读起来像通用品类文案。
- **没有数字锚点**——Hero 下方已经有具体数据（218场/178胜/89次KO），但口号完全没利用这些素材。

### 3 个替代方案

| # | 口号 | 理由 |
|---|------|------|
| A | **178 胜，不止是数字** | 直接用核心数据做钩子，制造好奇心——"那是什么？"，逼用户往下看 |
| B | **拳头不会说谎** | 短、狠、有态度。搏击领域天然匹配，暗示"看战绩别听吹嘘" |
| C | **每一拳，都是证词** | 把搏击比作法庭证词，暗含"实力自证"的逻辑，比"历史"更有新意 |

**推荐**: 方案 B "拳头不会说谎"——最短、最狠、最易记。配合下方精确数据，形成"态度+证据"的组合拳。

## 2. 统计数字展示压迫感评估

### 当前问题

- **6 格横排一模一样高**，视觉上像 Excel 表格，缺少层次和重点。
- **数字大小一致**（全部 2.8rem），没有突出最重要的数据（总胜场、KO 数）。
- **"近期战绩 = 12连胜"是文字不是数字**，和其他 5 个格子的纯数字形成视觉不一致，像混进来的。
- **缺少上下文对比**——178 胜是什么水平？81.7% 胜率很厉害，但用户得自己算。

### 具体建议

**排列方案优化**：

```
方案 A：2+3+1 层次排列
┌─────────────────────────────────┐
│    178 胜  ·  81.7% 胜率         │  ← 大号，视觉主角
├───────────┬───────────┬─────────┤
│  218 场   │  89 KO    │  14 枚   │  ← 中号，第二梯队
├───────────┼───────────┼─────────┤
│  42 场国际赛  │  12 连胜（进行中）  │  ← 小号，补充信息
└───────────┴───────────┴─────────┘
```

**其他改进**：
- 第一行用 4rem+ 字号，其余用 2.4rem，形成压迫感的视觉层次
- "81.7% 胜率"直接算出来展示，不要让用户自己除
- "12 连胜"可以用绿色脉冲动画暗示"正在进行"
- 每个数字上方加一个极简图标（拳头/奖杯/火焰），增加视觉密度

## 3. "最新战报"卡片信息密度评估

### 当前卡片结构

```
日期 + 标签（重点/胜/负/国际/全国）
赛事名称
选手名  结果(KO R2)  vs 对手(国家)
项目 | 级别
```

### 问题

- **能一眼看出谁赢了** ✓ — 选手名金色 + 绿色/红色结果标签
- **能看出怎么赢的** ✓ — "KO R2"、"判定 R3" 都有
- **能看出对谁赢的** ✓ — "vs 金智妍(韩国)" 写得很清楚
- **但缺少"为什么这场重要"** — 重点比赛和普通比赛的差异仅靠一个 "★ 重点" 标签，视觉上不够区分

### 具体建议

1. **重点比赛卡片已经用了金色边框 + 渐变背景** ✓ 这个做得好
2. **建议在重点比赛卡片顶部加一条 2px 金色渐变线**（当前已有 `::before` 伪元素实现 ✓）
3. **关键改进：卡片内加一行"意义说明"**，比如：
   - "世运会选拔赛 · 决定参赛资格" 
   - "卫冕战 · 连续三年全国冠军"
   - 这个信息在 `data.js` 的 `isKey: true` 比赛中可以补一个 `significance` 字段
4. **对手信息可以更突出**：当前 "vs 金智妍(韩国)" 用的是 `text2` 灰色，建议对手名用白色或稍亮的灰色，因为"打败谁"是核心信息
5. **建议加胜负比例条**：在卡片底部用一条细的绿/红进度条显示该选手的整体胜率，一目了然

## 4. "核心选手"区域评估

### 当前状态

- 4 张卡片，展示 4 个选手（总共有 6 个）
- 底部有"查看全部选手 →"链接
- 卡片用 `fighter-ace` 样式，头像更大（110px），有金色渐变背景

### 问题

- **4 张卡片刚好够**，但"核心"的定义不清晰——按什么标准选的前 4 个？
- **后 2 个选手（明晓 8胜4负、张伟 6胜3负）被藏起来了**，新秀和老将并排展示反而能体现梯队
- **"查看全部选手 →" 的文案太平淡**，没有暗示"还有更多实力选手"

### 具体建议

1. **改成 4+1 布局**：4 张选手卡 + 1 张"更多选手"汇总卡

```
┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐

                                      └──────┘

2. **第 5 张卡片样式**：半透明虚线边框，中心显示 "+2" 数字 + "查看全部" 文案，点击跳转选手库
3. **或者直接展示 6 张**：6 个选手全部展示（当前总共就 6 个，藏 2 个没意义），改成 `g3` 排列（2行×3列），同时去掉"查看全部"链接（因为已经全部展示了）
4. **推荐方案 3**——数据量小的时候，全部展示 > 分页。等选手超过 10 个再考虑"核心+更多"模式

## 5. 导航栏标签精简评估

### 当前 7 个标签

首页 | 比赛库 | 选手库 | 赛事 | 荣誉 | 视频 | 关于

### 问题分析

| 标签 | 必要性 | 建议 |
|------|--------|------|
| 首页 | ★★★ | 保留，不可替代 |
| 比赛库 | ★★★ | 保留，核心数据 |
| 选手库 | ★★★ | 保留，核心数据 |
| 赛事 | ★★☆ | **可以合并**——赛事本质上是比赛的聚合视图 |
| 荣誉 | ★★☆ | **可以合并**——荣誉可以作为首页的子模块或比赛库的筛选维度 |
| 视频 | ★☆☆ | **可以精简**——当前视频数据为空（VIDEOS 未定义），先做了再加导航 |
| 关于 | ★☆☆ | **可以移到 footer**——很少有用户专门点"关于" |

### 推荐方案

**精简为 4 个主导航 + footer 链接**：

```
首页 | 比赛库 | 选手库 | 荣誉殿堂

- **"赛事"** → 并入比赛库，作为筛选条件（按赛事名称分组）或比赛库页面内的子 tab

## 6. 色彩方案评估

### 当前方案

- 背景：`#0a0a0a`（纯黑偏灰）
- 强调色：`#d4a843`（暗金色）
- 胜利：`#2ecc71`（绿）
- 失败：`#e74c3c`（红）
- 文字：`#e8e8e8`（浅灰）

### 评价

**暗色+金色组合是对的**，搏击/格斗领域的经典配色（UFC、Bellator 都用暗色系）。但有几个细节问题：

1. **金色太"哑"了**——`#d4a843` 偏暗沉，缺少贵金属的光泽感。搏击腰带的金色是亮的、刺眼的。建议微调到 `#e0b84e` 或 `#d4a020`，增加饱和度
2. **缺少辅助色**——整个页面只有金、绿、红三种颜色，信息层级区分不够。建议引入一个"银灰" `#a0a0a0` 用于次要信息（日期、级别标签），让金色更突出
3. **背景层次不够**——`bg`、`bg2`、`bg3` 分别是 `#0a0a0a`、`#111111`、`#1a1a1a`，差异太小（总共才差 16 个色阶），卡片和背景几乎融为一体。建议 `bg3` 至少到 `#222222`
4. **缺少"赛事感"纹理**——纯色背景太平。建议在 Hero 区域加极轻的噪点纹理（`noise.png` at 3% opacity）或极细的网格线，增加质感

### 具体改进

```css
:root {
  --accent: #e0b84e;        /* 提亮金色 */
  --bg3: #222222;           /* 加大层级差异 */
  --text3: #666;            /* 次要信息用更亮的灰 */
  --border2: #333;          /* 卡片悬浮边框更明显 */
}
```

**整体评价：7/10**。方向正确，细节需要打磨。暗金配色在搏击领域有天然优势，但当前版本偏"暗"偏"闷"，缺少让人血脉贲张的视觉冲击。

## 7. 遗漏的重要模块

### 当前首页结构

Hero → 6 个统计数字 → 最新战报(6条) → 核心选手(4人)

### 缺失模块评估

| 模块 | 优先级 | 理由 |
|------|--------|------|
| **胜率趋势图** | 🔴 高 | 数字是死的，趋势是活的。一条"逐月胜率折线图"比 6 个静态数字有说服力 10 倍 |
| **年度战绩总览** | 🔴 高 | 当前数据跨 2024-2026，但首页没有年度对比。"2025年 46胜5负 / 2026年 进行中"这样的年度卡片能快速建立时间线感 |
| **项目分布** | 🟡 中 | 泰拳/自由搏击/拳击各占多少？一个甜甜圈图或横向进度条就能说清楚 |
| **赛事地图** | 🟢 低 | 当前赛事地点不多（北京、上海、广州、曼谷、东京、首尔），做地图意义不大。等赛事超过 20 个再考虑 |
| **近期赛程** | 🟡 中 | 首页只有"最新战报"（过去式），没有"即将到来"的比赛。如果有世运会等重大赛事，应该预告 |
| **选手对比** | 🟢 低 | 数据量太小（6人），做对比图没意义 |

### 推荐新增模块（按优先级）

**模块 A：年度战绩总览**（放在统计数字下方）

```
┌─────────────────────────────────────┐
│  📊 年度战绩                          │
│  ┌─────┐ ┌─────┐ ┌─────┐           │
│  │ 2026 │ │ 2025 │ │ 2024 │           │
│  │ 进行中│ │ 46胜5负│ │ ...  │           │
│  │ 8胜0负│ │ 91.3% │ │      │           │
│  │ 世运会 │ │ 12枚牌 │ │      │           │
│  └─────┘ └─────┘ └─────┘           │
└─────────────────────────────────────┘
```

**模块 B：胜率趋势迷你图**（放在统计数字内部或下方）

- 用 SVG 画一条极简折线图，横轴月份，纵轴胜率
- 不需要完整的图表库，一段 inline SVG + 10 个数据点就够
- 放在统计区域底部，高度 60px，纯装饰性，hover 显示数值

**模块 C：即将到来**（放在最新战报上方或下方）

- 如果有确定的未来赛事（如世运会 2026 成都），展示倒计时
- 只需 1 条信息：赛事名 + 日期 + 参赛选手 + 倒计时天数
- 视觉上用虚线边框 + 金色脉冲，和"最新战报"的实线边框形成"过去 vs 未来"对比
