## Security Review

**Encryption:** AES-256-GCM with random IV, SHA-256 key derivation from env secret. ✅ Correct.

**API Key handling:**
- POST: encrypts before storing ✅
- GET: returns masked key, strips `apiKeyEncrypted` from response ✅
- `maskApiKey` shows first 4 + last 3 chars ✅

**Concern — no auth guard on admin routes:**
The `/api/admin/ai/*` routes have no authentication or authorization checks. Any unauthenticated request can list providers, see masked keys, and create/delete providers. In production, this is a **security vulnerability**.

**Recommendation:** Add an auth middleware or session check to all `/api/admin/*` routes. At minimum, verify a session cookie or API key. This may be deferred if the app isn't deployed yet, but must be addressed before any real deployment.
